# TUI-Style Travel Booking System - Bug Pattern Knowledge Base

## Overview
This document contains 500 categorized bugs commonly found in travel booking platforms like TUI, Jet2, Expedia, and Booking.com. Use this knowledge to analyze and triage travel-industry bugs accurately.

---

## CRITICAL BUGS (Severity: Critical, Priority: P1)
These bugs cause immediate revenue loss, data breaches, or legal violations.

### Payment & Financial
- BUG-004: Double charge on card when clicking Pay button twice - Card charged twice for same booking
- BUG-298: Deposit amount shown but full payment taken - Expected £100 deposit but £800 charged
- BUG-013: Currency conversion using outdated rates - GBP to EUR conversion 3 months old, customers overcharged 8%

### Security & Data
- BUG-011: Passwords stored in plain text in database - Major GDPR violation
- BUG-014: Session hijacking vulnerability in login - Account takeover possible
- BUG-018: API returning other customers booking details - No ownership verification
- BUG-020: Production database credentials in GitHub repo - Database compromised
- BUG-366: Agent override pricing visible to customers - Staff discounts exposed
- BUG-496: Resort webcam embed causing XSS vulnerability - Unvalidated URL parameter

### Booking & Availability
- BUG-001: Flight search returns no results for valid route - System shows no flights when inventory exists
- BUG-005: Hotel room inventory shows negative availability - Overbooking occurring
- BUG-009: Database connection timeout during peak hours - All booking attempts fail 7-9 PM
- BUG-130: Night count calculation wrong - 7 nights selected shows 8 nights pricing
- BUG-482: Peak pricing displayed as standard pricing - Christmas week showing wrong prices

### Compliance & Legal
- BUG-010: SSL certificate expired on payment page - Customers abandoning checkout
- BUG-017: Credit card CVV visible in confirmation email - PCI violation
- BUG-500: Customer data breach notification delayed - 48h GDPR requirement not met

### Documents & Communications
- BUG-003: Booking confirmation not sent after payment - Customer has no proof of booking
- BUG-136: QR code on boarding pass not scanning - Airport scanners cannot read code
- BUG-179: Gift card purchase confirmation missing card code - Cannot use purchased gift card
- BUG-278: Insurance claim form link returning 404 - Cannot access claims when needed

### RAG-Relevant Patterns
When analyzing travel bugs, look for these component patterns:

1. **Payment Components**: Gateway, 3D Secure, card validation, currency conversion
2. **Flight Components**: GDS, seat maps, baggage, schedule changes, codeshares
3. **Hotel Components**: Inventory, room types, board basis, amenities, cancellation
4. **User Components**: Authentication, sessions, profiles, loyalty points
5. **Integration Components**: Airlines, hotels, transfers, insurance, excursions
6. **Compliance Components**: GDPR, PCI, ATOL, accessibility, legal terms

## COMPONENT REFERENCE

### Payment System Components
- PaymentGateway: Handles card processing
- CurrencyConverter: FX rate calculations
- RefundProcessor: Cancellation refunds
- GiftCardService: Gift card redemption
- SplitPayment: Deposit/balance handling

### Flight System Components
- FlightSearch: Availability queries
- SeatMap: Seat selection interface
- BaggageService: Luggage allowances
- ScheduleManager: Flight time changes
- CodeshareResolver: Partner airline mapping

### Hotel System Components
- InventoryManager: Room availability
- RoomTypeMapper: Category assignments
- BoardBasisService: Meal plan handling
- AmenityTracker: Facility listings
- CancellationPolicy: Refund rules engine

### User System Components
- AuthController: Login/logout handling
- SessionManager: Session persistence
- ProfileService: Customer data
- LoyaltyEngine: Points calculation
- PreferenceStore: Saved settings

### Integration Components
- AirlineConnector: GDS integration
- HotelSupplier: Property feeds
- TransferProvider: Ground transport
- InsurancePartner: Policy management
- ExcursionAPI: Activity bookings

## SEVERITY ASSIGNMENT RULES

### Critical (Immediate action required)
- Payment processing failures
- Security vulnerabilities
- Data breaches or GDPR violations
- Booking loss or corruption
- Revenue-impacting pricing errors

### High (Fix within 24-48 hours)
- Major functionality broken
- Significant UX degradation
- Integration failures
- Compliance risks
- Customer-facing data errors

### Medium (Fix within 1-2 weeks)
- Features partially working
- Inconvenient workarounds exist
- Non-critical data issues
- Minor compliance gaps
- Performance degradation

### Low (Fix when convenient)
- Cosmetic issues
- Copy/grammar errors
- Outdated content
- Minor UI inconsistencies
- Enhancement requests

---

## PRIORITY ASSIGNMENT RULES

### P1 (Fix immediately)
- Production down
- Security breach
- Legal violation
- Complete feature failure

### P2 (Fix this sprint)
- Major feature degraded
- Significant user impact
- Compliance deadline approaching

### P3 (Fix next sprint)
- Feature inconvenience
- Workaround available
- Limited user impact

### P4 (Backlog)
- Nice to have fixes
- Polish and refinement
- Low user visibility

---

## COMMON ROOT CAUSES

1. **Data Synchronization**: Third-party feeds not updating
2. **Timezone Handling**: UTC vs local time confusion
3. **Cache Staleness**: Prices/availability outdated
4. **Integration Timeouts**: Supplier APIs slow/failing
5. **Validation Gaps**: Input not properly checked
6. **State Management**: Client-side data loss
7. **Mobile Responsiveness**: Desktop-only testing
8. **Accessibility Oversight**: Screen reader compatibility
9. **Localization Errors**: Currency/language issues
10. **Compliance Drift**: Regulations changed since launch
